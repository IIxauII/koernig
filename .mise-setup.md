# mise Configuration

This project uses [mise](https://mise.jdx.dev/) (formerly rtx) to manage development tool versions.

## Tools Managed by mise

- **Node.js**: 22.12.0 (specified in `mise.toml`)
- **pnpm**: 10.22.0 (specified in `mise.toml`)

## Setup

1. Install mise if you haven't already:
   ```bash
   curl https://mise.run | sh
   ```

2. Activate mise in your shell:
   ```bash
   mise activate
   ```

3. Install the project's tools:
   ```bash
   mise install
   ```

This will automatically install the correct versions of Node.js and pnpm as specified in `mise.toml`.

## MCP (Model Context Protocol) Integration

mise provides an MCP server that allows AI assistants to query information about your development environment. This enables AI assistants to:

- Query installed tools and versions
- List available tasks in your project
- Access environment variables from mise configuration
- View mise configuration structure

### Setting up MCP with Claude Desktop

To enable mise MCP integration with Claude Desktop, add the following to your Claude configuration:

**macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`  
**Windows**: `%APPDATA%\Claude\claude_desktop_config.json`  
**Linux**: `~/.config/claude/claude_desktop_config.json`

```json
{
  "mcpServers": {
    "mise": {
      "command": "mise",
      "args": ["mcp"],
      "env": {
        "MISE_EXPERIMENTAL": "1"
      }
    }
  }
}
```

After adding this configuration and restarting Claude Desktop, the AI assistant will be able to:
- Query your installed tools and versions
- List available tasks in your project
- Access environment variables from your mise configuration
- View your mise configuration structure

### MCP Resources Available

The mise MCP server exposes the following resources:

- `mise://tools` - Lists all tools managed by mise (Node.js, pnpm, etc.)
- `mise://tasks` - Shows all available mise tasks
- `mise://env` - Displays environment variables defined in mise configuration
- `mise://config` - Provides information about mise configuration

For more information, see the [mise MCP documentation](https://mise.jdx.dev/mcp.html).

## Benefits

- **Consistent Environment**: All developers use the same tool versions
- **Automatic Activation**: Tools are automatically activated when entering the project directory
- **AI Integration**: MCP server enables AI assistants to understand your development environment
- **Version Pinning**: Tool versions are locked in `mise.toml` for reproducibility
